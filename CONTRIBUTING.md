# Contributing to Stethoscope for Networks

Thank you for considering contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check if the issue already exists in the [Issues](https://github.com/YOUR_USERNAME/stethoscope-for-networks/issues) tab
2. If not, create a new issue with a clear title and description
3. Include steps to reproduce the problem if applicable
4. Add relevant labels

### Submitting Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Add tests if applicable
5. Commit your changes (`git commit -m 'Add some feature'`)
6. Push to the branch (`git push origin feature/your-feature-name`)
7. Open a Pull Request

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Comment complex logic

### Testing

- Test your changes thoroughly before submitting
- Include example scripts if adding new features

## Questions?

Feel free to open an issue for questions or discussions.
